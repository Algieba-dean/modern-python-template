## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

We strongly encourage you to report security vulnerabilities through GitHub's **Private Vulnerability Reporting**.

### Option 1: GitHub Private Reporting (Recommended)

1. Go to the **Security** tab of this repository.
2. Click on **Advisories** in the left sidebar.
3. Click the green **Report a vulnerability** button.
4. Fill in the details. Only repository maintainers will see this report.

### Option 2: Email

If you prefer email, or if the private reporting feature is unavailable, please email us at:
**algieba.king@gmail.com**

## What to Expect

- You will receive a response within **48 hours**.
- We will handle your report with strict confidentiality.
- We will keep you informed of our progress towards resolving the problem.
- We will credit you (if you wish) once the vulnerability is fixed and publicly disclosed.

## Scope

We are interested in vulnerabilities that could affect the integrity or availability of the library, such as:

- Arbitrary Code Execution (ACE)
- Deserialization vulnerabilities (e.g., loading malicious model files)
- Sensitive data exposure

Thank you for keeping our community safe!
